Class ChainedBoxProperties | Methods
 
Packagebe.nascom.flash.physics
Classpublic class ChainedBox

ChainedBox, though not entirely using "real" physics, mimics a square box suspended by chains. Assign a Sprite instance (commonly a MovieClip on stage) with the current positions on stage set to where it will fall from. It will -not- interact with (ie: bounce off) other objects on stage.


Example
The following example gives chained box behaviour to Sprite instances on the stage (box, chain1 and chain2 are assumed to be on stage already): 
  import be.nascom.flash.physics.ChainedBox;
  
  var chained : ChainedBox = new ChainedBox(box, 300);  // create instance to animate box for a drop of 300 pixels
  chained.setChain1(chain1);        // assign first chain (optional)
  chained.setChain2(chain2);        // assign second chain (even more optional)
  chained.start(0, 10, 0);         // start animation, dropping it with no delay and with a slight initial momentum towards the right

durationproperty 
duration:Number  [read-write]

The maximum duration in frames of the total animation. The box will stop being animated after this time, regardless if the box has come to a standstill or not. This is so computations don't run after completion and that the box will stop fidgetting after a minimal amount of movement. Some experimentation might be required to set the right value.
